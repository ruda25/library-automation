반복 업무를 자동화로 바꾼 과정

아이디어 도출부터 Claude Code로 직접 구현하기까지

"내 업무 중 '이거 또 해야 하나…' 싶은
반복 작업이 있으신가요?" 제가 직접 반복 업무를 발견하고, 해결책을 찾고, 직접 자동화를 구현한 과정을 공유합니다.

개발 경험이 없어도 됩니다.
Apps Script를 한 번도 써본 적 없는 HR 담당자가 클로드코드와의 대화만으로 자동화 시스템을 완성한 이야기입니다.

📖
기존 운영 방식

전자도서관은 크루 입사와 동시에 LDAP과 동일한 ID/비밀번호로 회원 등록을 일괄 처리해왔습니다.

교보문고 플랫폼에서 임시비밀번호 생성이나 개별 알림(메일·문자 등) 기능을 제공하지 않아,
크루들이 별도 절차 없이 즉시 이용할 수 있도록 취할 수 있는 현실적인 운영 방법이었습니다.
🔍
문제 발견 — 알림 메일 한 통에서 시작됐습니다

2026년 2월, 월 대여 한도 150만원 달성 안내 메일을 받으면서 이상한 낌새를 포착했습니다.

①

로우 데이터 확인

대여 현황 파일을 열어보니 업무와 무관한 도서가 여러 아이디로 반복 대여된 패턴이 눈에 띄었습니다.

②

당사자 확인

의심 아이디를 특정해 해당 크루에게 직접 문의하자 "전자도서관에 로그인한 적 없다"는 답변을 받았습니다.
수년 전부터 본인이 아닌 누군가가 계정을 사용해온 사례도 확인되었습니다.

③

원인 파악

교보문고 측에 확인한 결과 외부 해킹이나 시스템 버그가 아닌 것으로 확인되었습니다.
LDAP과 동일한 계정 정보가 사실상 예측 가능한 상태였던 것이 근본 원인이었습니다.

같은 시기, 정보보안 조직으로부터 현행 로그인 계정 설정 방식의 보안 취약점에 대한 공식 보완 요청도 받았습니다.
문제를 더 이상 미룰 수 없는 상황이었습니다.
🤔
해결책 탐색 — 교보문고가 줄 수 있는 것의 한계

교보문고 측에 세 가지 해결 방안을 요청했습니다. 돌아온 답변은 한 가지였습니다.

교보문고의 제안 : 생년월일 6자리로 전 회원 비밀번호 일괄 초기화

검토해보니 이 방법도 한계가 명확했습니다.

한계
  • 생년월일은 여전히 예측 가능
  • 개인별 알림 기능 없음
  • 최초 로그인 시 강제 변경 기능 없음
  • 플랫폼 자체 개발 지원 불가
진짜 필요한 것
  • 개인별 무작위 비밀번호
  • 본인에게만 직접 전달
  • 받는 즉시 변경 유도
  • 담당자 혼자 처리 가능한 방법
💡
아이디어 전환점 — 플랫폼이 못 하면 내가 만든다

플랫폼에 없는 기능을 기다리거나 인프라 조직의 개발 지원을 요청하는 대신,
이미 갖고 있는 도구(구글 시트)로 직접 구현할 수 있다는 것을 떠올렸습니다.

이미 갖고 있는 것: Google Sheets, Google 계정, Apps Script(이름만 알고 있는 상태)
필요한 것: 무작위 비밀번호 생성 로직 + 개인 메일 자동 발송

두 가지를 연결하면 플랫폼의 한계와 무관하게 담당자가 직접 운영할 수 있는 자동화 흐름이 만들어질 것이라 생각했습니다.
문제는 Apps Script를 한 번도 써본 적이 없다는 것이었습니다.

🤖
Claude Code로 구현하기 — 코딩 언어가 아닌 업무 언어로

Apps Script 경험이 전혀 없는 상태에서 Claude Code에 상황을 설명하는 것부터 시작했습니다.
기술 용어가 아닌 담당자 입장의 업무 흐름으로요.

📋

무엇을 설명했는가

교보문고 플랫폼의 한계 (임시비밀번호 생성 기능 없음, 개별 알림 기능 없음)
원하는 결과 (개인별 무작위 비밀번호 생성 + 본인에게만 메일 발송)
현재 갖고 있는 도구 (구글 시트, 구글 계정)

🔄

어떻게 진행했는가

Claude Code가 Apps Script 코드를 만들어주면 직접 적용해보고,
안 되는 부분은 어떤 부분이 왜 안 되는지 다시 말로 설명하는 과정을 반복했습니다.
코드를 읽는 게 아니라 결과를 보고 피드백하는 방식이었습니다.

✅

결과

타 부서·개발 지원 없이 혼자 완성했습니다.
"해결하고 싶은 게 무엇인지 명확히 알고 있으면" 구현할 수 있다는 것을 확인했습니다.

정확한 프롬프트를 기억하지 못해도 됩니다. 중요한 건 문제를 언어로 정리하는 능력이었습니다.
🔁
같은 방식을 또 썼습니다 — 미승인 계정 일괄 처리

이번 개선 과정에서 지난 2월 미승인 상태로 잠궈두었던 계정들이 생각났습니다.
문의가 인입될 때마다 하나씩 처리하기 번거로운 상황이었고, 같은 접근 방식을 다시 적용했습니다.

문제: 미승인 계정 다수 존재 → 문의 인입 때마다 처리 시 담당자 반복 작업 발생
해결: Apps Script로 일괄 승인 전환 + 임시비밀번호 설정 + 자동 메일 안내를 한 번에 처리

한 번 패턴을 익히고 나면 다음 문제에도 빠르게 적용할 수 있습니다.
"이번에도 Claude Code에 상황을 설명하면 되겠다"는 판단 자체가 빨라졌습니다.

⚙️
완성된 자동화 프로세스

Google Sheets + Apps Script를 활용해 임시비밀번호 발급 → 개인 메일 자동 안내 흐름을 구현했습니다.

1

요청 접수

Google Sheets에 LDAP 아이디를 입력합니다.
계정발급비밀번호초기화 유형별 탭으로 구분 관리합니다.

2

임시비밀번호 자동 생성

메뉴 버튼 클릭 한 번으로 개인별 무작위 임시비밀번호를 자동 생성합니다.
대문자·소문자·숫자·특수문자 조합 12자리 (혼동 문자 제외).

3

사이트 수기 등록

담당자가 전자도서관 관리 사이트에서 생성된 임시비밀번호를 직접 등록합니다.

4

체크박스 체크 → 메일 자동 발송

등록 완료 후 시트의 체크박스를 체크하면 해당 크루에게 안내 메일이 즉시 자동 발송됩니다.
발송 완료 여부와 일시가 시트에 자동 기록됩니다.

🔒
보안 강화 결과

이번 개선은 카카오 HR 담당자 측의 운영 프로세스 변경과 교보문고 측의 시스템 개발이 함께 이루어진 결과입니다.

구분변경 내용
카카오 (운영) 임시비밀번호를 개인별 무작위 생성 후 본인에게만 메일로 개별 안내
교보문고 (시스템) 임시비밀번호로 최초 로그인 시 비밀번호 변경 필수 처리되도록 개발
두 변경이 맞물려 임시비밀번호 안내 → 본인만 수령 → 로그인 즉시 개인 비밀번호로 교체되는 구조가 완성되어,
예측 가능한 계정 정보를 통한 도용 가능성을 구조적으로 차단하게 되었습니다.
보안 강화

예측 가능한 계정 정보 제거
어뷰징 차단
최초 로그인 시 비밀번호 변경 유도로
도용 가능성 원천 차단
운영 효율화
메일 작성·발송 자동화로 담당자 반복 업무 제거
이력 관리
발송 여부와 일시가 시트에 자동 기록되어 추적 가능
🗺️
내 업무에 적용하기

문제의 형태는 달라도, 반복되는 운영 업무를 자동화로 바꾸는 사고 흐름은 같습니다.

1

반복을 발견한다

"이거 또 해야 하나" 싶은 순간을 흘려보내지 않기.

2

제약을 파악한다

플랫폼이 할 수 있는 것과 없는 것, 지원받을 수 있는 것과 없는 것을 먼저 확인합니다.

3

원하는 결과를 언어로 정리한다

기술 용어 없이, 담당자 입장에서 "이런 상황인데, 이런 결과가 필요하다"를 문장으로 씁니다.

4

Claude Code에 물어본다

코드를 몰라도 됩니다. 결과를 보고 안 되는 부분을 다시 설명하는 과정을 반복합니다.
